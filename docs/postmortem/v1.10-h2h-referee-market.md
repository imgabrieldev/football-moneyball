# Pitch — v1.10.0: H2H + Referee + Market Blending

## Problema

Modelo atual (v1.9.0, 48 features, Brier 0.65) ainda não usa:

1. **Head-to-head history** — rivalidades têm padrões claros (ex: Palmeiras tem vantagem histórica vs Botafogo). Zero features H2H.
2. **Referee tendencies** — Sofascore expõe + temos 99.8% coverage (468/469 partidas). Árbitros variam de 2.8 a 5.3 cards/game. Afeta probabilidade de expulsão e consequentemente de gol.
3. **Market probs (Betfair/Pinnacle)** — Pinnacle é referência de mercado eficiente. Temos 20 partidas com odds de 39 casas. Modelo operando sem olhar pra linha = ignorar informação do mercado.

**Target:** Brier 0.65 → <0.58.

## Solução

### Parte A: Head-to-Head Features (5 features novas)

Computar das últimas 5 partidas entre times (em qualquer competição):

- `h2h_home_win_rate` — % vitórias time_casa no H2H
- `h2h_away_win_rate` — % vitórias time_visitante no H2H
- `h2h_draw_rate` — % empates
- `h2h_home_goals_avg` — média gols casa
- `h2h_away_goals_avg` — média gols visitante

### Parte B: Referee Features (3 features novas)

Usando referee_stats join via match_stats.referee_id:

- `ref_cards_per_game` — tendência de cartões do árbitro designado
- `ref_strictness` — quantil (0.0 frouxo, 1.0 rígido) na distribuição
- `ref_home_advantage` — viés pro mandante (cards_home/cards_away nas últimas 10)

### Parte C: Market Blending (pós-processamento)

Não é feature — é blending no output final:

```
p_final = α · p_modelo + (1-α) · p_market_devigged
```

- Pega closing line da Betfair ou Pinnacle
- Remove vig (normaliza pra somar 1.0)
- Blend com α=0.55-0.70 (tunado via backtest)
